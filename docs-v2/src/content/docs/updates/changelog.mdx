<AccordionItem title="v2 alpha (anchor-next)" defaultOpen>

`anchor-lang-v2` is the next-generation runtime, built on [pinocchio](https://github.com/anza-xyz/pinocchio) and `#![no_std]{:rs}` by default. It produces an order of magnitude smaller binaries and fewer CU per instruction than v1. See the [v2 docs](/docs/v2/) for the full reference and migration guide. The v1 (`lang/`) code in this repository remains the stable, published path.

#### Major changes

- lang: New `anchor-lang-v2` crate built on pinocchio with no `<'info>{:rs}` lifetimes ([#4393](https://github.com/otter-sec/anchor/pull/4393)).
- lang: `Account<T>{:rs}` is zero-copy by default and requires `T: Pod{:rs}`. Variable-length data uses `BorshAccount<T>{:rs}`.
- lang: Drop `Pubkey{:rs}` for `Address{:rs}` (drop-in 32-byte type from pinocchio).
- lang: Trait-based extensibility via `AnchorAccount{:rs}` and `AccountConstraint<A>{:rs}` lets third-party crates ship new account primitives without forking the derive ([#4414](https://github.com/otter-sec/anchor/pull/4414)).
- lang: Macro-time PDA bump precomputation. Seeds that are byte literals get the canonical bump baked in as a `const{:rs}`, skipping the runtime search.
- lang: `#[event]{:rs}` defaults to wincode (borsh-wire-compatible). `#[event(bytemuck)]{:rs}` for fixed-size events emits one disc plus a memcpy.
- lang: Mask-based duplicate-account detection ([#4419](https://github.com/otter-sec/anchor/pull/4419)).
- lang: `#[pod_wrapper]{:rs}` attribute for safe enum-to-pod conversion that validates discriminants on equality and conversion.
- lang: `Slab<H, Item>{:rs}` zero-copy ledger / event-log accounts.
- lang: `Nested<T>{:rs}` for sharing `#[derive(Accounts)]{:rs}` validation across instructions.
- lang: Deprecate `has_one{:rs}` for the more general `address{:rs}` constraint ([#4426](https://github.com/otter-sec/anchor/pull/4426)).
- lang: Pod helpers `PodU64{:rs}`, `PodI128{:rs}`, `PodBool{:rs}`, `PodVec<T, MAX>{:rs}` with alignment-1 layout for clean `#[repr(C)]{:rs}` packing.

#### Tooling

- cli: New `$ <blue>anchor</blue> debugger` TUI for stepping through SBF instructions per test, built on the same register traces as `$ <blue>anchor</blue> test <dim>--profile</dim>`.
- cli: `$ <blue>anchor</blue> test <dim>--profile</dim>` produces per-test register-trace flamegraphs under `target/anchor-v2-profile/`.
- testing: New `anchor-v2-testing` crate wraps `LiteSVM{:rs}` with optional register-trace capture (`profile{:rs}` feature).

#### Optimizations

- Skip on-curve check for program-owned PDAs (~1,000 CU saved per verify).
- `const-rent` feature folds `Rent::get(){:rs}` to a compile-time constant (~85 CU per `create_account{:rs}`).
- `guardrails{:rs}` feature gates runtime safety nets behind a default-on flag. Disabling drops ~300 bytes binary, ~1-2 CU per account.
- `account-resize` feature gates `realloc_account{:rs}` and the pinocchio resize hook.
- Lazy-cached `remaining_accounts(){:rs}`. Instructions that don't touch it pay zero cost.
- Typed `CpiHandle{:rs}` enables `invoke_signed_unchecked{:rs}` safely. One fewer runtime check per CPI.
- Drop `#[inline(never)]{:rs}` on cold error helpers.

#### Spl

- spl: New `anchor-spl-v2` crate with `Mint{:rs}`, `TokenAccount{:rs}`, `InterfaceAccount{:rs}` for both the Token Program and Token Extension Program.
- spl: Token-2022 extension CPI re-exported via `pinocchio_token_2022`.

#### Bug fixes

- v2: Fix four correctness bugs surfaced by fuzzing and add a `PodVec{:rs}` validation API ([#4431](https://github.com/otter-sec/anchor/pull/4431)).
- v2: Validate discriminant in `PodWrapper{:rs}` `PartialEq{:rs}` ([#4442](https://github.com/otter-sec/anchor/pull/4442)).
- v2: Deserialize instruction data once, share parsed args ([#4440](https://github.com/otter-sec/anchor/pull/4440)).
- v2: Cover account wrapper checks and `remaining_accounts{:rs}` paths ([#4445](https://github.com/otter-sec/anchor/pull/4445)).
