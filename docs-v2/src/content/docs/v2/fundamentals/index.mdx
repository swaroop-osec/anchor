Every Anchor program touches `#[program]{:rs}`, `Context<T>{:rs}`, `#[derive(Accounts)]{:rs}`, PDA validation, IDL emission, and CPI.

If you are building your first program, read the pages in the order shown. Program structure covers where code goes. Accounts, context, and validation cover what happens before a handler runs. PDAs cover derived addresses. The IDL is what clients see. CPI is how one program calls another.
